# Frontend Architecture & Screens Documentation

Our Flutter frontend is designed specifically for children, maintaining an engaging, distraction-free environment while invisibly collecting medical-grade telemetry.

## App State & Providers
- **`AuthProvider`**: Manages user sessions, JWT tokens, and login states (`lib/providers/auth_provider.dart`).
- **`ApiService`**: Handles all network traffic with the Django backend including authentication, game session creation, and result submission (`lib/services/api_service.dart`).

## Core Screens
### 1. HomeScreen (`lib/screens/home_screen.dart`)
- **Purpose**: Entry point after login. Displays dashboard cards.
- **Implementation**: Uses an animated layout with Lottie/Rive animations (like `space_dog_4.riv`). Contains navigation routes to the Game Hub and Report screens via `ChildSelectionScreen`.

### 2. ChildSelectionScreen (`lib/screens/child_selection_screen.dart`)
- **Purpose**: Lets a parent/teacher select which child is currently playing or being reviewed.
- **Implementation**: Fetches the list of children tied to the account via `ApiService.getChildren`. Uses a boolean flag `isForReport` to decide whether tapping a child routes to the `GameHubScreen` or `ReportScreen`.

### 3. ReportScreen (`lib/screens/profile/report_screen.dart`)
- **Purpose**: Displays the aggregated screening results (Low, Moderate, High risk) for the selected child.
- **Implementation**: 
    - Fetches data using `ApiService.getChildReport`.
    - Generates dynamic UI cards with specific color codings (Green = Low, Orange/Yellow = Moderate, Red = High Risk) based on the backend API response.
    - Includes a comprehensive textual summary algorithm generated by the backend.

## Game Screens & Frameworks
Games are built using the **Flame Engine** for precise 60fps rendering and input telemetry collection.

### 1. Star Tracer (Dysgraphia Module)
- **Path**: `lib/game/star_tracer_game.dart`, `lib/screens/games/star_tracer_screen.dart`
- **Implementation**:
    - **Flame Game**: The user is tasked with tracing a dashed letter path. 
    - **Telemetry Collection**: Intercepts `onPanStart`, `onPanUpdate`, and `onPanEnd` events to record lists of coordinate maps: `{'x': dx, 'y': dy, 't': timestamp}`.
    - **LetterGuideComponent**: (`lib/game/components/letter_guide_component.dart`) Contains mathematical `Path` definitions for all 26 alphabets (A-Z) mapped to screen space.
    - **Flow**: Selects 5 random alphabets per session.

### 2. Echo Explorers (Dyslexia Module)
- **Path**: `lib/game/echo_explorers_game.dart`, `lib/screens/games/echo_explorers_screen.dart`
- **Implementation**:
    - **Setup**: Evaluates phonological awareness (rhyming/matching sounds).
    - **Audio**: Uses `audioplayers` package to play target phonemes/words.
    - **Interaction**: Displays 4 floating "Bubble" components. 
    - **Telemetry Collection**: Captures the Boolean accuracy (did they tap the right rhyming word) and `ReactionTime` (DateTime difference from when audio finished playing to when they tapped).

### 3. Monster Munch (Dyscalculia Module)
- **Path**: `lib/screens/games/monster_munch/`
- **Implementation**: Custom painted widgets and animations (Asset-light approach).
- **SubitizingScreen**: Tests rapid number sense. Shows a set of items (1-9) for a brief second. The child must input the correct number. 
- **ComparisonScreen**: Tests magnitude logic. Shows two changing numbers, and the child must feed the larger number to the monster. 
- **Telemetry Collection**: Records response values, expected values, success boolean, and reaction times using standard Flutter `GestureDetector` logic embedded in an `AssessmentEngine`.
