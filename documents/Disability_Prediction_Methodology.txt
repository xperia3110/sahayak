# Disability Prediction Methodology

### The Core Problem: How to predict without an ML Dataset?

In an ideal Artificial Intelligence scenario, we would have thousands of hours of gameplay telemetry mapped against clinically verified diagnoses of Dyslexia, Dysgraphia, and Dyscalculia. We would train Convolutional Neural Networks (CNNs) on raw drawing paths or Random Forests on reaction times.

**However, without access to standard clinical datasets, how do we determine if a child has a disability using the values available?**

### The Solution: Medical Heuristics & Deterministic Proxies

Instead of relying on black-box ML pattern matching, our backend utilizes **Heuristic Expert Systems** that encode established medical and psychological literature thresholds directly into deterministic algorithms.

We act as an automated version of a clinical psychologist administering standard paper-and-pencil diagnostic tests.

#### 1. Proxies for Dysgraphia (Star Tracer)
Medical literature indicates dysgraphic children have significantly higher deviations from target lines and higher variation in pen pressure/speed. 
- **Raw Values Available**: Arrays of `(X, Y, Time)` touch points.
- **Processing**: The `KinematicAnalyzer` processes this raw array to calculate `Acceleration` (Jitter Proxy) and `RMSE` (Spatial Accuracy Proxy). 
- **Determination**: Without a dataset, we set theoretical medical thresholds. If spatial error > 150px and jitter variance is high, deterministic logic flags the child as High Risk, mimicking a clinician noting a "shaky, inaccurate trace."

#### 2. Proxies for Dyslexia (Echo Explorers)
- **Raw Values Available**: Audio cue finish time, touch time (`Reaction Time`), selected answer vs correct answer (`Accuracy`).
- **Processing**: The rapid naming and phonological matching tasks are digital versions of the CTOPP (Comprehensive Test of Phonological Processing). 
- **Determination**: Instead of a dataset, we use known developmental baselines (e.g., standard neurotypical reaction time for phoneme matching is ~1.5s - 2.5s). If a child consistently takes `>4.0s` and has `<60%` accuracy, our algorithm flags them, removing the need for an ML training set to tell us this is anomalous.

#### 3. Proxies for Dyscalculia (Monster Munch)
- **Raw Values Available**: Array of `(ReactionTime, isCorrect, NumberShown)`.
- **Processing**: We look for the psychological "Subitizing limit". 
- **Determination**: By comparing average reaction times of items 1-4 vs items 5-9, our backend doesn't need a dataset to "learn" the pattern. We strictly code the logic: If `Slope(ReactionTime_Small) == Slope(ReactionTime_Large)`, then `Subitizing == False`. This directly translates to a high dyscalculia risk according to cognitive psychology.

### Real-time Report Aggregation
When a professional requests a report (`GET /api/children/<id>/report/`), the system doesn't run a predictive neural network. Instead it:
1. Fetches the latest deterministic Risk Scores (`0.0 - 1.0`).
2. Averages them per category.
3. Groups the outputs into standard diagnostic bins:
   - `< 0.3` = Low Risk (Neurotypical)
   - `0.3 - 0.6` = Moderate Risk (Monitor)
   - `> 0.6` = High Risk (Refer for clinical evaluation)
   
Using standard heuristics makes the system transparent, trackable, and highly effective even without thousands of rows of training data.
